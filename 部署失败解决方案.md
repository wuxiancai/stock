## 解决方案

### 方案一：手动修复服务（推荐）

#### 1. 修复PostgreSQL
```bash
# 检查PostgreSQL状态
sudo systemctl status postgresql

# 如果未运行，启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 配置PostgreSQL认证
sudo -u postgres psql -c "ALTER USER postgres PASSWORD 'postgres';"

# 修改认证方式
sudo sed -i "s/#listen_addresses = 'localhost'/listen_addresses = '*'/" /etc/postgresql/*/main/postgresql.conf
sudo sed -i "s/local   all             postgres                                peer/local   all             postgres                                md5/" /etc/postgresql/*/main/pg_hba.conf

# 重启PostgreSQL
sudo systemctl restart postgresql
```

#### 2. 修复Redis
```bash
# 安装Redis服务器
sudo apt-get update
sudo apt-get install -y redis-server

# 启动Redis服务
sudo systemctl start redis-server
sudo systemctl enable redis-server

# 测试Redis连接
redis-cli ping
```

#### 3. 重新运行部署
```bash
# 清理之前的虚拟环境（可选）
rm -rf venv

# 重新运行部署脚本
./deploy_robust.sh
```

### 方案二：使用Docker部署（备选）

### 方案三：简化部署脚本

创建一个专门针对Ubuntu的简化部署脚本：

```bash
#!/bin/bash
# Ubuntu专用部署脚本

set -e

echo "🚀 Ubuntu系统部署开始..."

# 1. 安装PostgreSQL和Redis
sudo apt-get update
sudo apt-get install -y postgresql postgresql-contrib redis-server

# 2. 配置PostgreSQL
sudo systemctl start postgresql
sudo systemctl enable postgresql
sudo -u postgres createdb stock_trading || true
sudo -u postgres psql -c "ALTER USER postgres PASSWORD 'postgres';" || true

# 3. 启动Redis
sudo systemctl start redis-server
sudo systemctl enable redis-server

# 4. 设置Python环境
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install psycopg2-binary
pip install -r requirements.txt

# 5. 配置环境变量
cp .env.example .env

# 6. 初始化数据库
python app/init_db.py

echo "✅ 部署完成！"
echo "启动命令: source venv/bin/activate && uvicorn app.main:app --reload"
```

## 立即修复步骤

### 第一步：修复PostgreSQL认证
```bash
# 切换到postgres用户并设置密码
sudo -u postgres psql -c "ALTER USER postgres PASSWORD 'postgres';"

# 修改认证配置
sudo nano /etc/postgresql/*/main/pg_hba.conf
# 将 "local all postgres peer" 改为 "local all postgres md5"

# 重启PostgreSQL
sudo systemctl restart postgresql
```

### 第二步：安装并启动Redis
```bash
sudo apt-get install -y redis-server
sudo systemctl start redis-server
sudo systemctl enable redis-server
```

### 第三步：测试连接
```bash
# 测试PostgreSQL
psql -h localhost -U postgres -d postgres -c "SELECT 1;"

# 测试Redis  
redis-cli ping
```

### 第四步：重新运行部署
```bash
# 激活虚拟环境
source venv/bin/activate

# 手动创建数据库
createdb -h localhost -U postgres stock_trading

# 运行初始化脚本
python app/init_db.py
```
