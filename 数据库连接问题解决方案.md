# 数据库连接问题解决方案

## 问题描述

用户在运行 `python3 main.py` 时遇到数据库连接失败错误：

```
File "/home/ubuntu/stock/app/database/init_db.py", line 14, in init_database 
    raise Exception("数据库连接失败")
```

## 问题原因

1. **PostgreSQL服务未启动**
2. **数据库和用户未创建**
3. **配置文件中的数据库连接信息不正确**
4. **缺少必要的数据库驱动**

## 解决方案

### 🚀 方案一：一键部署（推荐）

运行完整的部署脚本，自动处理所有初始化：

```bash
chmod +x deploy_complete.sh
./deploy_complete.sh
```

### 🔧 方案二：手动初始化

#### 1. 启动PostgreSQL服务

**macOS:**
```bash
brew services start postgresql
```

**Linux:**
```bash
sudo systemctl start postgresql
```

**Docker:**
```bash
docker run -d --name postgres \
  -p 5432:5432 \
  -e POSTGRES_PASSWORD=postgres \
  postgres:13
```

#### 2. 初始化数据库

```bash
chmod +x init_database.sh
./init_database.sh
```

#### 3. 配置环境变量

复制并编辑配置文件：
```bash
cp .env.example .env
# 编辑 .env 文件，确保数据库配置正确
```

#### 4. 安装依赖

```bash
pip install psycopg2-binary
pip install -r requirements.txt
```

### 📋 配置说明

在 `.env` 文件中确保以下配置正确：

```env
# 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_NAME=stock_db
DB_USER=stock_user
DB_PASSWORD=stock_password
DATABASE_URL=postgresql://stock_user:stock_password@localhost:5432/stock_db
```

### 🔍 验证步骤

1. **检查PostgreSQL服务**：
   ```bash
   pg_isready -h localhost -p 5432
   ```

2. **测试数据库连接**：
   ```bash
   psql -h localhost -p 5432 -U stock_user -d stock_db -c "SELECT version();"
   ```

3. **运行应用程序**：
   ```bash
   python3 main.py
   ```

### 🐳 Docker部署（备选方案）

### 🛠️ 故障排除

#### 问题1：PostgreSQL连接被拒绝
```bash
# 检查服务状态
brew services list | grep postgresql  # macOS
systemctl status postgresql           # Linux

# 重启服务
brew services restart postgresql      # macOS
sudo systemctl restart postgresql     # Linux
```

#### 问题2：用户认证失败
```bash
# 重新创建用户
sudo -u postgres psql -c "DROP USER IF EXISTS stock_user;"
sudo -u postgres psql -c "CREATE USER stock_user WITH PASSWORD 'stock_password';"
sudo -u postgres psql -c "ALTER USER stock_user CREATEDB;"
```

#### 问题3：数据库不存在
```bash
# 重新创建数据库
sudo -u postgres createdb -O stock_user stock_db
```

### 📞 获取帮助

如果仍有问题，请检查：

1. **日志文件**：`logs/app.log`
2. **PostgreSQL日志**：通常在 `/var/log/postgresql/`
3. **环境变量**：确保 `.env` 文件配置正确
4. **网络连接**：确保端口5432未被占用

### 🔄 重置数据库

如需完全重置数据库：

```bash
# 删除现有数据库
sudo -u postgres dropdb stock_db

# 重新初始化
./init_database.sh
```
