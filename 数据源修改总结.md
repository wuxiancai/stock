# 股票交易系统数据源修改总结

## 修改背景
根据用户要求，将超出 Tushare 积分限制的接口改为使用 AKshare，并确保所有实时数据都通过 AKshare 获取。

## 积分限制说明
- **用户当前积分**: 2000 积分
- **需要 3000 积分的接口**: share_float (限售股解禁)
- **需要 5000 积分的接口**: concept (概念股分类)、concept_detail (概念股列表)

## 主要修改内容

### 1. 数据源策略调整

#### 修改前
```
历史数据获取策略：TUshare (主) → AKshare (备用) → 缓存数据 (兜底)
实时数据获取策略：AKshare (主) → TUshare (备用) → 缓存数据 (兜底)
```

#### 修改后
```
历史数据获取策略：TUshare (主，2000积分限制) → AKshare (备用) → 缓存数据 (兜底)
实时数据获取策略：AKshare (主，免费无限制) → 缓存数据 (兜底)
特殊数据获取策略：
- 概念股数据：AKshare (TUshare concept 接口需要5000积分)
- 限售股解禁：AKshare (TUshare share_float 接口需要3000积分)
- 所有实时行情：AKshare (免费且更新频率高)
```

### 2. TushareService 类修改

#### 移除的方法
- `get_concept()` - 概念股分类数据 (需要5000积分)
- `get_concept_detail()` - 概念股列表数据 (需要5000积分)

#### 添加的注释说明
```python
# 注意：concept 和 concept_detail 接口需要5000积分，已移除
# 改用 AKshare 的概念股接口：
# - ak.stock_board_concept_name_em() 获取概念板块名称
# - ak.stock_board_concept_cons_em() 获取概念板块成份股
```

### 3. AKshare 配置增强

#### 新增的方法
- `get_stock_restricted_release_queue_sina()` - 获取限售股解禁队列 (替代 Tushare share_float)
- `get_stock_restricted_release_summary_em()` - 获取限售股解禁统计

### 4. 数据服务层修改

#### 概念股数据获取方法
- **修改前**: 优先使用 Tushare，失败时回退到 AKshare
- **修改后**: 仅使用 AKshare 获取概念股数据

#### 新增限售股解禁数据方法
```python
async def get_restricted_release_data(self) -> Optional[Dict[str, pd.DataFrame]]:
    """获取限售股解禁数据 - 使用 AKshare (Tushare share_float 接口需要3000积分)"""
```

### 5. 任务调度修改

#### 概念股更新任务
- **修改前**: 使用 Tushare 的 `get_concept()` 和 `get_concept_detail()`
- **修改后**: 使用 AKshare 的 `get_stock_board_concept_name_em()` 和 `get_concept_stocks()`

#### 新增限售股解禁更新任务

### 6. API 接口文档更新

#### 新增数据源说明
```
### 4.1 数据源说明
- **实时数据**: 全部使用 AKshare 获取，免费且更新频率高
- **历史数据**: 优先使用 TUshare (2000积分内接口)，超出积分限制的接口使用 AKshare
- **概念股数据**: 使用 AKshare (TUshare concept 接口需要5000积分)
- **限售股解禁**: 使用 AKshare (TUshare share_float 接口需要3000积分)
```

#### 章节编号调整
- 所有后续章节编号相应调整 (4.2 → 4.3, 4.3 → 4.4, 等等)

## 修改效果

### 1. 积分使用优化
- 避免使用超出2000积分限制的 Tushare 接口
- 概念股和限售股解禁数据改用免费的 AKshare 接口
- 所有实时数据使用 AKshare，节省 Tushare 积分

### 2. 数据获取稳定性
- 实时数据完全依赖 AKshare，避免积分限制导致的服务中断
- 保持历史数据的高质量 (使用 Tushare 2000积分内接口)
- 增加限售股解禁数据的获取能力

### 3. 成本控制
- 最大化利用免费的 AKshare 资源
- 合理使用有限的 Tushare 积分
- 确保系统在积分限制下正常运行

## 注意事项

1. **数据字段差异**: AKshare 和 Tushare 的数据字段名可能不同，需要在数据处理时进行字段映射
2. **更新频率**: AKshare 的实时数据更新频率可能与 Tushare 不同，需要调整缓存策略
3. **数据质量**: 需要监控 AKshare 数据质量，必要时进行数据清洗
4. **接口稳定性**: AKshare 作为免费服务，需要做好容错处理

## 后续优化建议

1. **数据验证**: 建立 AKshare 和 Tushare 数据的交叉验证机制
2. **缓存策略**: 针对不同数据源调整缓存时间和策略
3. **监控告警**: 建立数据源可用性监控和告警机制
4. **数据备份**: 重要数据建立多重备份机制
